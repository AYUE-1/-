## 一、项目概述

**爪印**是一个基于 Vue 3 + Spring Boot 的全栈宠物领养信息平台，为流浪动物救助机构与爱心领养人士搭建沟通桥梁。平台涵盖宠物领养、流浪救助、社区互动、养宠科普、公益志愿、信任评价等功能模块，支持三种角色（普通用户、救助机构、管理员）的协作运营。

### 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue 3 (Composition API) | 3.5 |
| 构建工具 | Vite | 8.0 |
| UI 组件库 | Element Plus | 2.14 |
| 状态管理 | Pinia | 3.0 |
| 路由 | Vue Router | 4.6 |
| HTTP 客户端 | Axios | 1.18 |
| 后端框架 | Spring Boot | 3.2.0 |
| 安全认证 | Spring Security + JWT | jjwt 0.12.3 |
| ORM | MyBatis-Plus | 3.5.5 |
| 数据库 | PostgreSQL / H2 (dev) | — |
| API 文档 | Knife4j (Swagger) | 4.4 |
| Java | JDK | 17 |

## 四、功能模块详解

### 4.1 用户认证 (`/api/auth`)

- **注册**: 支持选择普通用户或救助机构身份，用户名3-50字符，密码6位以上
- **登录**: JWT 无状态认证，Token 有效期 24 小时
- **JWT 载荷**: userId, username, role
- **角色验证**: 普通用户不可注册为管理员

### 4.2 宠物领养 (`/api/pets`, `/api/adoptions`)

**宠物管理:**
- 宠物 CRUD（名称、品种、年龄、性别、体型、健康状态、疫苗、绝育、性格描述等）
- 分类筛选（猫咪/狗狗/其他）
- 多维度搜索（关键词、分类、性别、体型）
- 排序方式（最新发布、最多浏览）
- 图片上传与管理

**领养流程（两阶段审核）:**

```
用户提交申请 → 管理员初审 → 宠物主人终审 → 领养完成
   (PENDING)  (ADMIN_APPROVED)  (APPROVED)    (COMPLETED)
                    ↘ 拒绝          ↘ 拒绝
                  (REJECTED)    (OWNER_REJECTED)
```

**领养申请表字段:** 真实姓名、身份证号、联系电话、居住地址、住房类型、养宠经验、家人同意、职业、收入范围、申请理由

### 4.3 流浪救助 (`/api/rescue`)

- **发布求助**: 标题、描述、位置（经纬度+地址）、动物类型、紧急程度（普通/紧急/危重）
- **救助地图**: 地图可视化展示附近求助信息
- **认领机制**: 志愿者可以认领救助帖，状态流转 PENDING → PROCESSING → RESCUED → CLOSED
- **跟进记录**: 认领后可添加救助进展
- **距离排序**: 基于 Haversine 公式计算

### 4.4 养宠测评 (`/api/assessment`)

- **12 道测评题目**，覆盖 7 个维度：居住空间、时间充裕度、养宠经验、预算、家庭构成、活跃程度、宠物偏好
- **提交后生成测评报告**，推荐最适合的宠物类型
- 支持匿名测评（无需登录）

### 4.5 养宠科普 (`/api/articles`)

- 文章分类：猫咪、狗狗、通用、Checklist
- 标签系统：支持多标签关联
- 发布审核：普通用户投稿需管理员审核，救助机构/管理员自动发布
- 养宠前 Checklist

### 4.6 社区互动 (`/api/community`)

- **帖子类型**: 经验分享 (SHARING)、寻宠启事 (LOST_FOUND)
- **互动功能**: 点赞/取消点赞、嵌套评论（支持回复）
- **审核机制**: 管理员可审核/屏蔽帖子

### 4.7 公益板块 (`/api/welfare`)

- **志愿者申请**: 申请即通过，填写技能特长、可服务时间、所在区域
- **志愿者排行榜**: 按救助数量排序，展示金银铜牌
- **志愿者主页**: 公开详情页，展示救助统计、救助动物列表、公益事迹
- **公益数据**: 认证志愿者人数、累计救助动物数

### 4.8 机构工作台 (`/api/shelter`)

- **数据仪表盘**: 发布宠物数、在架数、待审核申请数、已领养数
- **宠物管理**: 卡片式管理，支持下架/上架/删除
- **领养审核**: 表格视图，展开查看申请人详情，同意/拒绝
- **机构认证**: 提交认证材料 → 管理员审核 → 展示认证标识
- **公开主页** (`/shelter/:id`): 展示机构信息、在架宠物、领养统计

### 4.9 管理后台 (`/api/admin`)

| 模块 | 功能 |
|------|------|
| 仪表盘 | 宠物总数、待审核申请、已领养宠物、注册用户数 |
| 宠物管理 | 全平台宠物管理（搜索、状态变更、编辑、删除） |
| 领养审核 | 一审（通过转发布者确认 / 拒绝） |
| 用户管理 | 用户列表、启用/禁用账号 |
| 科普文章 | 文章管理、审核、标签管理 |
| 社区审核 | 帖子审核、屏蔽 |
| 举报管理 | 查看举报、处理/忽略 |
| 认证审核 | 机构认证通过/拒绝（含拒绝理由） |

### 4.10 其他功能

| 模块 | 说明 |
|------|------|
| **收藏** | 收藏/取消收藏宠物，查看收藏列表 |
| **评论** | 宠物详情页嵌套评论（支持回复） |
| **信任体系** | 领养完成后的双向评价（1-5星+文字），机构专属黑名单 |
| **健康档案** | 宠物健康记录（疫苗/驱虫/就医/日记），到期提醒 |
| **通知系统** | 新申请、审核结果、认领等自动通知 |
| **实用工具** | 附近宠物医院地图、养宠成本计算器、急救指南 |
| **举报系统** | 举报违规内容/用户，管理员处理 |

## 十、关键设计说明

### 10.1 两阶段领养审核

这是平台最复杂的业务流程：

1. 用户提交领养申请 → 状态 `PENDING`
2. 管理员一审 → `ADMIN_APPROVED`（通过，转发布者确认）或 `REJECTED`（拒绝）
3. 宠物主人（发布者）二审 → `APPROVED`（同意）或 `OWNER_REJECTED`（拒绝）
4. 领养完成 → `COMPLETED`

### 10.2 JWT 无状态认证

- Token 存储在客户端 localStorage
- 每个请求通过 `Authorization: Bearer <token>` 传递
- Token 载荷包含 userId、username、role
- 过滤器 `JwtAuthenticationFilter` 解析 Token 并设置 SecurityContext
- **注意**: Token 中的角色在签发时固定，修改数据库角色后需重新登录

### 10.3 H2 开发数据库

开发环境使用 H2 文件数据库 (`jdbc:h2:file:./data/pet_adoption`)，无需安装 PostgreSQL。数据文件存储在 `pet-adoption-backend/data/` 目录。数据库结构由 `SchemaMigration.java` 在启动时自动创建和升级。

### 10.4 三种角色独立工作区

| 角色 | 工作区 | 入口 |
|------|--------|------|
| 普通用户 | 前台页面 | 顶部导航栏 |
